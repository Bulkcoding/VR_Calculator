# VR 리밸런싱 기본 정보

## 계산식

- **거치식 VR 계산식**: `V2 = V1 + pool/G`
- **적립식 VR 계산식**: 거치식 VR 계산식 + 적립금
- **인출식 VR 계산식**: 거치식 VR 계산식 - 인출금
- 파라미터 저장시 현재 사이클을 거치식 or 적립식 or 인출식에서 어떤 방식으로 진행할건지 정할수 있게 해준다.

## 사이클

- 사이클은 2주마다 한번씩 돌아간다.
- 사이클 기록의 컬럼은 다음과 같다.
  - 시작평가금
  - 마지막 평가금
  - V
  - 최소밴드
  - 최대밴드
  - 시작 pool
  - 마지막 pool
- **시작평가금**: 이전사이클의 마지막평가금. 따라서 첫 사이클에서는 해당주식수 × 전날종가
- **마지막평가금**: 사이클 마지막 날의 종가 × 주식수(매도,매수 했던것도 합쳐야함)
- **V값**:
  - 1사이클에는 1사이클의 시작평가금을 넣어주면된다.
  - 2사이클부터는 각 계산식(거치식 or 적립식 or 인출식)에 따라서 계산값을 넣어주면 된다.
- **최소/최대밴드**: V값을 기준으로 ±10, ±15, ±20% 3개중 선택할수 있도록 한다.
  - 예) ±10% 선택 → 최소밴드: V값 × 0.9 / 최대밴드: V값 × 1.1
  - (±기준만 선택하면 자동으로 값을 기재해주도록 하면되서 텍스트를 직접 수정은 막도록 한다.)
- **시작 pool**: 이번 사이클의 pool값
- **마지막 pool**: 매도, 매수 후 이번 사이클 pool값

## 계산식 ++ (심화과정)

심화과정의 계산식
- `V2 = 21 + pool/G + (E - V1) / (2√10)`
- E: 사이클의 '마지막 평가금'에 해당하는 값이다

## 매도 매수

- 매수는 현재 사이클의 pool의 **75%** 근처까지만 사용할수 있도록 매수 개수를 제한하도록 함.
- 매도 매수는 각 매도, 매수 시점마다 1주씩 할수도 있고, 2주씩 할수도 있다. 사용자 정의를 통해 정하도록 한다.

## 표시 규칙

- 보유수량 / 매수·매도 후 누적수량은 **소수점 둘째 자리까지만** 표시한다. (예: `0.83` — `0.83000000000000001` 같은 부동소수점 노이즈 숨김)

## 매도 차수 제한

- 매도 차수 생성 시 다음 두 조건 중 하나라도 해당하면 그 차수는 만들지 않는다.
  1. 해당 차수의 **매도 단가가 1 미만** (주가 1 미만의 자산은 매도 분할 대상에서 제외)
  2. 매도 후 **남은 수량이 1주 미만** (예: 6.83주에서 1주씩 매도 시 마지막 7차 매도는 0.83주가 되므로 제외 → 6차까지만 생성)

## 매도 행 표시

- 각 차수 행은 `차수 / [단위 입력: N주] / @단가 / 누적수량 / 잔여Pool` 순으로 표시한다.
- `@단가`는 **자동 계산된 1주당 매도가격**(`maxBand / 매도전수량`)이며 직접 수정할 수 없다. 단위(주 수)만 수정 가능하다.
